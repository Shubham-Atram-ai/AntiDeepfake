## 3. Architectural Decisions

### Separation of Concerns

Each Milestone 2 component has a single, well-defined responsibility:

| Component | Single Responsibility |
|-----------|----------------------|
| `FGSMAttack` | Generate adversarial perturbations in embedding space |
| `compute_ssim` | Measure structural visual similarity |
| `compute_psnr` | Measure signal-level distortion in dB |
| `test_fgsm_pipeline.py` | Orchestrate the full pipeline — no business logic |

The pipeline script is intentionally kept thin: it delegates to `FaceDetector`,
`FGSMAttack`, and the metrics module. No adversarial logic lives in the pipeline
script itself.

### Frozen Model as Feature Extractor

`InceptionResnetV1` is frozen immediately after loading:

```python
for param in model.parameters():
    param.requires_grad_(False)
```

This ensures:
- No accidental fine-tuning.
- Gradient flow is restricted to the **input tensor only**.
- Runtime memory footprint is minimised (no optimizer state).

### Automatic Device Selection

`FGSMAttack(device="auto")` transparently selects CUDA when available and falls
back to CPU with no code change required. The pipeline script passes `device="auto"`
to ensure maximum hardware utilisation without user configuration.

### RGB Purity Guarantee

The entire pipeline operates exclusively in **RGB colour space** from load to
save:

```
load_image()        → RGB uint8
detect_and_crop()   → RGB uint8
face_tensor         → RGB float [0,1]
adversarial_tensor  → RGB float [0,1]
adversarial_face    → RGB uint8
cloaked_rgb         → RGB uint8
save_image()        → converts RGB → BGR internally → cv2.imwrite
```

This single-direction flow eliminates any risk of accidental double-conversion
("Smurf Face" blue tint artifact) that can occur when `cv2.COLOR_RGB2BGR` is
called redundantly.

## 4. FGSM — Fundamentals and Design

### What is FGSM?

The **Fast Gradient Sign Method** (Goodfellow et al., 2015) is a single-step
white-box adversarial attack. Given an input `X`, a loss function `L`, and a
model `f`, the adversarial example is:

$$X_{adv} = X + \varepsilon \cdot \text{sign}\left(\nabla_X \, L(f(X), y)\right)$$

The sign of the gradient indicates the direction of steepest ascent in loss,
and `ε` (epsilon) is the maximum per-pixel perturbation (L∞ budget).

### Why FGSM for Adversarial Cloaking?

FGSM is:
- **Fast:** Single forward + backward pass. No iterative optimisation.
- **Interpretable:** The perturbation is the sign of the gradient — no hyperparameters beyond `ε`.
- **Effective:** At `ε = 0.02`, consistently degrades recognition accuracy while remaining
  visually imperceptible.
- **Baseline-setting:** More powerful attacks (PGD, C&W) build on FGSM principles.

### Loss Function: Negative Cosine Similarity

FaceNet embeddings are L2-normalised vectors on the unit hypersphere. The
natural distance metric for unit vectors is **cosine distance**:

$$L = -\cos\theta = -\frac{e_{orig} \cdot e_{adv}}{\|e_{orig}\| \cdot \|e_{adv}\|}$$

Minimising this loss (via gradient ascent on the input) maximises the angle
between the original and adversarial identity vectors — directly targeting the
mechanism face-recognition systems use to identify individuals.

Alternative losses (MSE, Euclidean) were considered:

| Loss | Pros | Cons |
|------|------|------|
| **Negative cosine similarity** ✓ | Bounded [-1,1]; scale-invariant; matches unit-sphere geometry | None for this application |
| MSE | Simple | Not scale-invariant; sensitive to embedding magnitude |
| Negative Euclidean distance | Intuitive | Can be dominated by large-magnitude components |

## 5. Why FaceNet Requires an Embedding-Space Loss

Standard classification-based FGSM maximises cross-entropy loss w.r.t. a
class label. FaceNet produces no class probability output — it produces a
**512-dimensional embedding vector**. There is no softmax, no class index.

Applying classification FGSM to FaceNet would be nonsensical because:
1. There is no class label `y` to target.
2. Cross-entropy loss is undefined without a categorical distribution.

The embedding-space approach is correct because:
- Face recognition systems compute identity by comparing embedding vectors
  (e.g. using a cosine similarity threshold).
- Maximising embedding distance directly attacks the similarity computation
  that determines recognition.
- The loss remains fully differentiable through the model, enabling clean
  gradient computation.

## 6. Why Native PyTorch Was Selected

The project already uses PyTorch as its primary ML framework (via
`facenet-pytorch`). Native PyTorch FGSM was selected over alternatives
for the following reasons:

| Option | Decision | Reason |
|--------|----------|--------|
| **Native PyTorch** ✓ | Selected | Zero new dependencies; full control; transparent gradient flow |
| Adversarial Robustness Toolbox (ART) | Not used | High-level wrapper obscures gradient computation; embedding-space attacks require custom loss integration |
| Foolbox | Not used | Additional dependency; no meaningful advantage over native PyTorch for FGSM |
| CleverHans | Not used | TensorFlow-primary; PyTorch support is secondary |

Using `autograd` directly also makes the implementation **auditable** — the
gradient computation is visible in source code with no framework magic.

## 7. Image Reconstruction Design

### The Challenge

The FGSM attack operates on a 160 × 160 FaceNet input tensor. The original
image may be any resolution (e.g. 1920 × 1080). Reconstruction requires:

1. Converting the adversarial tensor back to uint8 RGB.
2. Resizing from 160 × 160 to the original bounding box dimensions.
3. Inserting the resized adversarial face exactly back into the original image.

### Bounding Box Clamping (Belt-and-Suspenders)

`FaceDetector.detect_and_crop()` already clamps bounding box coordinates
internally (Milestone 1, lines 140–143). The pipeline applies a **second
defensive clamp** before array slicing:

```python
x1 = max(0, min(int(bounding_box[0]), img_w - 1))
y1 = max(0, min(int(bounding_box[1]), img_h - 1))
x2 = max(x1 + 1, min(int(bounding_box[2]), img_w))
y2 = max(y1 + 1, min(int(bounding_box[3]), img_h))
```

This prevents out-of-bounds array slicing regardless of the bounding box
source — critical for future extension (e.g. loading bbox from JSON).

### Resize Strategy

Resizing uses **Lanczos** (LANCZOS4 in OpenCV) for both scale-up
(tensor prep) and scale-down (reconstruction). Lanczos is the standard
high-quality resampling filter for facial images — it minimises ringing
and aliasing artefacts.

## 8. Evaluation Metrics

### Structural Similarity Index (SSIM)

SSIM models the **human visual system's** sensitivity to structural changes.
It computes three sub-comparisons across a sliding window:

$$\text{SSIM}(x, y) = \frac{(2\mu_x\mu_y + C_1)(2\sigma_{xy} + C_2)}{(\mu_x^2 + \mu_y^2 + C_1)(\sigma_x^2 + \sigma_y^2 + C_2)}$$

- **Luminance term**: Compares average pixel intensity.
- **Contrast term**: Compares standard deviation of intensities.
- **Structure term**: Compares the normalised cross-correlation.

Range: `[-1.0, 1.0]`. A value of `1.0` indicates identical images.
Typical human perception threshold: `≥ 0.90`.

Implementation uses `skimage.metrics.structural_similarity` with:
- `channel_axis=2` — correct for HWC (Height × Width × Channels) arrays.
- `data_range=255` — correct for uint8 images.
- `win_size=7` — 7×7 pixel sliding window (default).

### Peak Signal-to-Noise Ratio (PSNR)

PSNR is the ratio of the maximum possible signal power to the noise power:

$$\text{PSNR} = 10 \cdot \log_{10}\!\left(\frac{\text{MAX}^2}{\text{MSE}}\right)$$

where `MAX = 255` for uint8 images and MSE is the mean squared error
between corresponding pixels.

| PSNR Range | Interpretation |
|------------|---------------|
| `> 40 dB` | Practically indistinguishable from original |
| `30 – 40 dB` | Minor visible noise on close inspection |
| `< 30 dB` | Noticeable distortion |
| `∞` | Identical images |

Identical images produce MSE = 0 → division by zero. This is handled
explicitly by detecting MSE = 0 and returning `float('inf')`.

## 9. Expected Behaviour at ε = 0.02

At `epsilon = 0.02` (2% of the normalised pixel range):

| Metric | Expected Value | Interpretation |
|--------|---------------|----------------|
| SSIM | 0.90 – 0.98 | Near-imperceptible perturbation |
| PSNR | 33 – 44 dB | Minor signal-level noise; typically undetectable |
| Cosine similarity | 0.70 – 0.95 | Significant identity vector displacement |
| L∞ perturbation | ≤ 0.02 (5.1/255) | Sub-threshold for most human observers |

> **Note on variability:** Exact values depend on the specific input image,
> face size, and FaceNet weight initialisation. High-contrast faces with fine
> facial details (e.g. facial hair, glasses) will typically yield larger SSIM
> drops for the same epsilon.

## 10. Known Limitations

### FGSM Is Single-Step

FGSM is a one-shot attack. While fast, it is less powerful than iterative
attacks (PGD, C&W). A well-adversarially-trained recognition model may
partially resist single-step FGSM perturbations.

### White-Box Assumption

FGSM requires gradient access to the target model. If the deployed
face-recognition system is a different model (e.g. ArcFace, DeepFace), the
perturbations generated against FaceNet may not transfer perfectly.
**Transferability** across models is an active research area.

### No Adaptive Refinement

The current implementation applies a uniform epsilon across all pixels.
Future implementations could use a **saliency mask** to apply larger
perturbations only in regions that contribute most to the embedding (e.g.
eyes, nose) while preserving other regions at higher quality.

### Preprocessing Round-Trip

The face crop is resized to 160 × 160 for FaceNet, perturbed, then resized
back to the original bbox dimensions. Each resize introduces minor interpolation
artifacts that slightly reduce SSIM and PSNR independently of the adversarial
perturbation.

### JPEG Compression

Saving as JPEG applies lossy compression, which can reduce the effectiveness
of the adversarial perturbation (JPEG compression has a smoothing effect that
partially cancels high-frequency perturbations). For strongest adversarial
effect, save outputs as PNG or use JPEG quality=100.

## 11. Suggested Next Steps — Milestone 3

### 3.1 Projected Gradient Descent (PGD) Attack

Replace single-step FGSM with multi-step PGD for stronger, more reliable
perturbations:

$$X^{t+1} = \text{clip}_{X,\varepsilon}\!\left(X^t + \alpha \cdot \text{sign}(\nabla_X L)\right)$$

### 3.2 Transferability Testing

Generate perturbations against FaceNet (VGGFace2) and evaluate whether they
transfer to other recognition models (ArcFace, DeepFace, InsightFace).

### 3.3 Adaptive Saliency Masking

Compute a gradient saliency map over the face region and apply larger
perturbations only to identity-critical regions (eyes, nose bridge) while
leaving background pixels unmodified.

### 3.4 Batch Processing CLI

Add a command-line interface that processes a directory of input images
and outputs a directory of cloaked images, with configurable epsilon and
device selection.

### 3.5 Adversarial Robustness Evaluation

Integrate ART (already in `requirements.txt`) to evaluate the protection
quality against certified defences (e.g. randomised smoothing, adversarial
training).

### 3.6 Web Interface

Extend `src/frontend/` with a Flask or FastAPI backend + minimal web UI
for drag-and-drop image cloaking through a browser.
